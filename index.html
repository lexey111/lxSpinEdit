Key features

  • Integers and floats are supported, both positive and negative.
  • Keyboard navigation (use Ctrl+↑ ↓ when input is focused).
  • Step x10 (use Shift key while clicking plus and minus).
  • Mousewheel support upon either input and slider.
  • Trackbar slider control.
  • Automatic range check and error indication.
  • Two kind of progress bar: fullsized and embedded.
  • Clear and useful API.
  • LESS style file with all the constants isolated — easy to customize.

Require

  • jQuery 1.8+

Valid for (i.e. tested in)

  • Chrome (webkit) 24+
  • Opera 12.15+
  • Firefox 20+
  • IE 10+
  • Safari 5+

How to use

LxSpinEdit is a runtime-creation kind of UI controls. This mean that you have to make new Javascript object to use it.

First step is prepare container for item(s) in html-file. Next, you need to call constructor explicitly to make and embed selector into prepared place.

Usual way to do this is something like (in the documentReady() function or later):

Usage example

    var cntr_opacity = new LxSpinEdit({
        container: item_container,
        id: 'cntr_opacity',
        title: 'Opacity',
        fixed: true,
        slider: true,
        always_use_wheel: true,
        min: 1,
        max: 100,
        step: 1,
        value: 100,
        measure: '%',
        percent_bar: true,
        callback_change: onDataChanged,
        callback_focus: showInitialProperties
    });

Properties list

Property Optional Type What does it mean
id No string Unique control ID.
container No string|object jQuery selector (jQuery object or text) of parent container for button.
title Yes string Label before the control.
hide_title Yes boolean Hide title or not, default: false
enabled Yes boolean Is control enabled or is it read-only, default: true
min No integer|float Minimum value. Min and max could be optional, i.e. "0", but only if slider is set to false.
max No integer|float Maximum value.
value No integer|float Current value.
fixed Yes boolean Use integers (true) or floats (false). Default: false
point_pos Yes integer Precision, i.e. number of digits after decimal point if fixed is false. Default: 1
step No integer|float Value that will be added/subtracted from current on plus/minus button clicking.
measure Yes string Measurement units, text after the control.
always_use_wheel Yes boolean When true, mouse wheel will interacts with control always (otherwise - only if control is focused). Default: false
slider Yes boolean Use or not dropdown slider (trackbar). Default: false
live_slider Yes boolean Call the callback function every time while dragging slider handle (or only when drag ends). Default: false
clear_after Yes boolean Add "clearfix" div after control so next control will starts from new line. Default: false
percent_bar Yes boolean Use or not percentage bar (inside control). Default: false
percent_full_size Yes boolean Use all the control background as percentage bar, otherwise — use ony thin bar at the bottom. Default: false
hide_buttons Yes boolean Hide plus and minus buttons while control has no focus. Default: false
callback_change Yes function onChange event callback. Function get control as this object and current value as parameter.
callback_focus Yes function onFocus event callback. Function get control as this object.

Change callback

Usage example:

  function onDataChanged(value) {
    // do nothing, append to console
    // LxSpinEdit control is passed here as 'this' parameter
    pseudo_console.log(this.id, value, this.properties.measure);
  }

Please take it into account: value is current value in the control while this.properties.value is initial value.
